# AI 产品经理面试复盘

## 工作流程

### 第一步：确认输入

向用户确认面试复盘的内容来源。不强求格式，只要能识别出"问题"和"回答"即可。

### 第二步：问题分类

读取 `references/assessment-framework.md`。

对每一组"问题-回答"，判断它属于哪个象限：

- **象限一**（自我介绍与转型动机、岗位选择）：问"我是谁/为什么转/为什么适合"
- **象限二**（产品能力与项目落地路径）：问"有没有产品基本功/能不能做成"
- **象限三**（AI能力与技术判断）：问"懂不懂AI接入方式和能力边界"
- **象限四**（AI发散认知与成长潜力）：问"有没有持续学习/趋势判断/成长空间"

判据：看问题的**意图**而不是字面。

### 第三步：逐题评估

对每个回答，对照 assessment-framework 中对应象限的评估矩阵，逐维度判断：

1. 标记 **🟢 做得好的**：引用矩阵中"优秀"列的匹配项
2. 标记 **🔴 需改进的**：引用矩阵中"存疑"列的匹配项
3. 检查 **通用失分**：具体性缺失/逻辑链断裂/自洽性存疑/深度不扛压

**评估原则：**
- 每个结论必须引用原始回答中的原文作为证据
- 宁严勿宽
- 回答质量评级：🟢优秀 / 🟡合格 / 🔴需改进

### 第四步：输出复盘文档

按 `references/output-template.md` 的**四部分结构**输出完整复盘文档。

**必须严格遵循四部分结构，不可自行调整：**

#### 第零部分：目标公司与岗位介绍

- 根据面试官在对话中的介绍整理公司背景、产品定位、业务线、团队文化、岗位画像
- 标注「非候选人回答，仅背景参考」

#### 第一部分：面试问题按类型罗列

- 仅罗列问题原文，不包含回答、不包含评估
- 按四象限分组
- 格式：编号 + 问题原文

#### 第二部分：按类型降序逐题评估

- 每个象限内，按回答质量从 🟢优秀 → 🟡合格 → 🔴需改进 的降序排列
- 每道题包含：问题原文 + 原始回答摘要 + 评估矩阵对照 + 质量评级

#### 第三部分：改进示范

- 针对 🔴需改进 和 🟡合格 的题目（🟢优秀 不在此列）
- 先对原始回答进行拆解（指出具体哪句话有什么问题）
- 再给出改进版回答，标注改进点
- 改进回答结构：因→做→果三层，基于候选人经历补充和重组
- 评估标准宁严勿宽：🟡合格意味着存在可提升空间，同样值得改进示范

#### 第四部分：候选人反问记录

- 记录候选人向面试官提出的所有问题和面试官的回答
- 标注「不纳入评分」
- 按提问顺序排列

### 第五步：保存文档

文档命名格式（**严格遵循，不可加任何额外描述**）：**YYYY-MM-DD-评分.md**

- 仅包含日期和评分，不添加公司名、轮次、"面试复盘"等任何前缀后缀
- 评分规则：🟢优秀→A / 🟡合格→B / 🔴需改进→C，可加 +/- 微调
- 信息不全无法评分时：使用 `N-A`
- 示例：`2026-05-09-B+.md`、`2026-05-09-N-A.md`

保存位置优先级：

### 第六步：概要汇报

文档生成后，用 5 行以内向用户汇报：
- 整体质量判断
- 最突出的 1-2 个优点
- 最需要改进的 1-2 个问题
